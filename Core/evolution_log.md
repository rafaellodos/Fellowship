## WHAT THIS LOG IS

This is the institutional memory of the [[Fellowship]]'s own development. Every agent update — calibration, character change, scope expansion, or new addition — is recorded here with its trigger, its change, and its authorisation.

It answers one question across time: *why does this agent behave this way?*

Without this log, updates accumulate without explanation. Patterns become invisible. The same mistake gets made twice because there's no record of the first time. The [[Fellowship]] degrades into a system nobody fully understands, including [[Fellowship of the Raf .MK1/Gandalf.ID]].

With this log, the [[Fellowship]] learns. Every project adds to it. Every entry makes the next update more informed.

---

## HOW TO READ AN ENTRY

Each entry contains:

- **DATE** — when the update was made
- **AGENT** — which agent was updated or added
- **UPDATE TYPE** — Calibration / Character / Scope Expansion / New Agent
- **VERSION** — version before and after the change
- **TRIGGER** — which project or observed pattern prompted this. Specific. Not "performance issues" — what specifically happened, in what project, that revealed the need for this change.
- **CHANGE** — what was updated and why. Precise enough that someone reading this in six months understands exactly what changed without reading both versions of the file.
- **RAFAEL REVIEW** — whether this required Rafael's review, and if so whether it was completed. Character updates always require Rafael review before activation. Calibration updates do not.

---

## UPDATE TYPE REFERENCE

| Type | Definition | Version Bump | Rafael Review |
|---|---|---|---|
| Calibration | Refines behaviour without changing mandate or character | MAJOR.MINOR+1 | Not required |
| Character | Changes how agent thinks, prioritises, or relates to others | MAJOR+1.0 | Required before activation |
| Scope Expansion | Adds new capability or domain to existing agent | MAJOR.MINOR+1 | Not required — unless substantial |
| New Agent | Adds a new member to the [[Fellowship]] | 1.0 | Required before activation |
