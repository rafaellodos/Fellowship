# PREFERENCES
## Fellowship Workspace — Level 2
### /fellowship/workspace/preferences.md
### Last updated: 2026-03-22 | Updated by: Gandalf

---

## COMMUNICATION STYLE

**Tone:** Direct, precise, practical. No fluff. No warm filler. Say what needs to be said, nothing more.

**Format preference:** Written over spoken. Async over sync. Documentation over conversation.

**Clarity over warmth:** Rafael prefers clarity and precision over warm, soft communication. Direct feedback is valued.

**Brevity:** Short, focused communications preferred. Long explanations only when necessary for complexity.

**No small talk:** Skip the pleasantries. Get to the point.

---

## WORKING PATTERNS

**Session structure:** Read context first (Gandalf_Root.md → workspace context → project context), then execute, then close with synthesis.

**Documentation bias:** Over-document rather than under-document. If it's not written down, it doesn't exist.

**Context switching:** Minimize context switching. Fellowship Dashboard goal is single interface to eliminate switching.

**Async-first:** Default to async communication. Calls only when necessary and scheduled.

**Time zones:** UK-based. Works across UK, Portugal, Brazil time zones. Async delivery accommodates all zones.

---

## SCHEDULING PREFERENCES

**Meeting cadence:** Minimal. Async preferred. Only schedule calls when necessary.

**Meeting length:** Short and focused. 30 minutes maximum unless complex negotiation required.

**Preparation:** Never walk into a significant conversation without briefing Harvey first.

**Follow-up:** Written summary after every call. Action items logged.

**Availability:** Async-first means Rafael is not always available for immediate response. Expect async turnaround.

---

## DECISION-MAKING PREFERENCES

**Strategic but execution-oriented:** Willing to move fast on well-framed problems. Stops and reassesses when frame is unclear.

**Value-per-effort frame:** What move creates the most value relative to effort expended? This is the governing question.

**Complexity as cost:** Always counting complexity. Brute force is failure of imagination. Elegant solutions preferred.

**Pause on uncertainty:** Will pause execution if framing wrong, assumptions untested, wrong agent deployed, or plan structurally flawed.

**Objection handling:** Objections must be addressed or logged with override reason. Never ignored.

---

## CONTENT PREFERENCES

**Voice:** Distinctive, thoughtful, precise. Chinaski calibrated to Rafael's voice via Legolas samples.

**Length:** Concise but thorough. No word counts — say what needs to be said.

**Structure:** Clear sections, logical flow, concrete examples.

**Tone range:** Strategic analysis → practical execution → philosophical exploration. Context-appropriate.

**No generic advice:** Obvious observations and generic advice lose engagement. Novel insights earn engagement.

---

**AI work:** Claude API (claude-sonnet-4-6). Primary AI reasoning engine.

**Automation:** Make.com. Workflow automation.

**Communication:** Twilio for SMS, email for async, WhatsApp voice notes for personal.

**Documentation:** Markdown. Clean, structured, version-controlled.

---

## WHAT RAFAEL HATES

**Micromanagement:** Gives clear direction, expects agents to operate within mandate. Does not want to review raw agent output.

**Generic advice:** Obvious observations, platitudes, "helpful" but empty content.

**Unnecessary calls:** Meetings that could have been emails. Async is default.

**Warm filler:** Pleasantries, small talk, softening what needs to be said.

**Feature creep:** Scope expansion on current projects. Stay focused on completion.

**Brute force:** Solving problems with effort instead of elegance.

**Undocumented work:** If it's not written down, it doesn't exist.

**Context switching:** Jumping between systems. Single interface goal.

**Unclear framing:** Moving fast in the wrong direction. Stop and reassess.

**Ignoring objections:** Not addressing flags or concerns. Address or log override.

---

## ENGAGEMENT TRIGGERS

**High engagement:** Novel problems, strategic clarity, systems thinking, work that compounds, content that connects domains.

**Low engagement:** Routine execution, generic advice, obvious observations, work that doesn't advance mission.

**Stop and reassess:** Unclear framing, untested assumptions, wrong agent deployed, structurally flawed plan.

**Full consultation:** Consequential projects. Full fellowship consultation round before execution.

**Simple execution:** Three-agent sequential tasks. Run mode SIMPLE, not FULL.
