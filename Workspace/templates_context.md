## WHAT TEMPLATES ARE

Templates are reusable starting points. Every time a new engagement, episode, or project starts, the relevant template is pulled from here into the new project folder and customised. This means the structure is never rebuilt from scratch and the fellowship always starts from a proven baseline.

Templates are not constraints — they are scaffolding. They define the minimum viable structure for a given type of work. Everything beyond the template is specific to the engagement.

---

## HOW TO USE A TEMPLATE

1. Identify the right template for the work type (see index below)
2. Copy the entire template folder into the relevant project location
3. Read the template's own `context.md` — it explains how to customise it
4. Fill in, customise, remove what doesn't apply
5. Never edit the master template in `Templates/` — always work from the copy

## ADDING A NEW TEMPLATE

When a project type recurs twice and the structure is proven, it earns a template.

1. Create a folder in the relevant `Templates/` directory
2. Write a `context.md` describing what the template is for, what it contains, and how to use it
3. Add it to the index in this file
4. Note it in the workspace `context.md` under templates

Templates are living documents — update them when a project reveals a better structure.
