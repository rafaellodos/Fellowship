# INTAKE CHECKLIST TEMPLATE

## PROJECT INTAKE CHECKLIST

Use this checklist when starting a new project to ensure all necessary information is captured.

### BASIC INFORMATION
- [ ] Project name follows naming convention (kebab-case, dated)
- [ ] Workspace identified (Rafael / AvenoirTech / Inspiracy / ThoughtExperiment / Labs)
- [ ] Project type defined (Client project / Internal project / Template / Documentation)
- [ ] Status set (PLANNING / ACTIVE / ON HOLD / COMPLETED)

### PROJECT DEFINITION
- [ ] Project purpose clearly stated
- [ ] Primary focus identified
- [ ] Project goals listed (at least 3)
- [ ] Deliverables defined
- [ ] Stakeholders identified

### CONTEXT
- [ ] Background information collected
- [ ] Constraints or limitations noted
- [ ] Success criteria defined
- [ ] Dependencies identified

### RESOURCES
- [ ] Agent access permissions defined
- [ ] Skills needed identified
- [ ] Timeline established
- [ ] Budget or resource requirements noted

### CLIENT INFORMATION (if applicable)
- [ ] Client name and contact information
- [ ] Client requirements documented
- [ ] Contract or agreement details noted
- [ ] Communication preferences established

### FOLDER SETUP
- [ ] context.md created and populated
- [ ] intake/ folder created with brief.md
- [ ] deliverables/ folder created
- [ ] communications/ folder created (if client project)
- [ ] Additional intake subfolders created as needed (research/, notes/, requirements/)

### NEXT STEPS
- [ ] Brief reviewed and approved
- [ ] Project kickoff scheduled (if applicable)
- [ ] Work plan created
- [ ] First deliverable identified
