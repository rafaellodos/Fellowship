# FELLOWSHIP PROJECT TEMPLATE - USER GUIDE

## OVERVIEW

This template provides a standardized structure for all projects within the Fellowship of the Raf system. It ensures consistency, clarity, and efficient project management across all workspaces.

## HOW TO USE THIS TEMPLATE

### Step 1: Copy the Template
When starting a new project, copy this entire folder structure to the appropriate workspace:
```
[workspace]/Projects/[YYYY-MM-DD-project-name]/
```

### Step 2: Customize the Project
1. Rename the project folder following naming conventions
2. Edit `context.md` with your project details
3. Copy `intake/project-brief_template.md` to `intake/brief.md`
4. Fill in the brief with your project information
5. Use `intake/intake-checklist_template.md` to ensure completeness

### Step 3: Start Work
- Add research materials to `intake/research/`
- Save notes to `intake/notes/`
- Document requirements in `intake/requirements/`
- Create deliverables in `deliverables/` following naming conventions
- Track communications in `communications/` (for client projects)

## TEMPLATE FILES

### Core Files
- **context.md** - Project overview and status
- **intake/project-brief_template.md** - Template for project briefs
- **intake/project-structure_guide.md** - Detailed structure guide
- **intake/intake-checklist_template.md** - Intake checklist

### Reference Files
- **deliverables/this-guide.md** - This user guide
- **deliverables/naming-conventions.md** - Naming convention reference
- **deliverables/agent-reference.md** - Agent access reference

## NAMING CONVENTIONS

### Projects
- Format: `YYYY-MM-DD-project-name`
- Style: kebab-case
- Example: `2026-03-22-fellowship-project-template`

### Deliverables
- Format: `[YYYY-MM-DD]_[descriptor]_[type].[ext]`
- Example: `2026-03-22_strategy-deck_proposal.pptx`
- Example: `2026-03-22_user-research_report.pdf`

### Folders
- Use lowercase with hyphens for multi-word folders
- Example: `research/`, `notes/`, `requirements/`

## WORKSPACE GUIDANCE

### Rafael (Personal)
- Use for personal projects, writing, ideas
- Can omit communications/ folder
- Focus on personal goals and deliverables

### AvenoirTech (Business)
- Client projects require communications/ folder
- Include client requirements in intake/
- Follow business deliverable naming conventions

### Inspiracy (Consulting)
- Client projects require communications/ folder
- Strategic analysis and consulting deliverables
- Emphasis on research and requirements

### ThoughtExperiment (Content)
- Use Episodes/ for YouTube content
- Focus on content development and production
- Can omit communications/ for internal content

### Labs (Technical)
- Use Experiments/ for AI testing
- Technical documentation and experiments
- Can omit communications/ for internal projects

## BEST PRACTICES

1. **Always start with the brief** - Fill in project-brief_template.md first
2. **Update context.md regularly** - Keep status and decisions current
3. **Follow naming conventions** - Consistent naming aids organization
4. **Use the checklist** - Ensure all information is captured
5. **Document decisions** - Record key decisions in context.md
6. **Organize intake materials** - Use subfolders for different types of inputs
7. **Version deliverables** - Use dates in deliverable names
8. **Track communications** - Keep client correspondence organized

## COMMON WORKFLOWS

### Starting a Client Project
1. Copy template to workspace/Projects/
2. Fill in brief with client information
3. Create context.md from brief
4. Gather requirements in intake/requirements/
5. Conduct research in intake/research/
6. Develop deliverables in deliverables/
7. Track all communications in communications/

### Starting an Internal Project
1. Copy template to workspace/Projects/
2. Fill in brief with project goals
3. Create context.md from brief
4. Gather research and notes in intake/
5. Develop deliverables in deliverables/
6. Omit communications/ folder

### Using the Fellowship
1. Identify which agents need access (see agent-reference.md)
2. Specify skills needed in brief
3. Reference workspace context.md for agent permissions
4. Follow agent protocols in core/fellowship.md
